**Project 3 — Summership 2026 | Vicharanashala Lab, IIT Ropar**

---

## What This Is

An automated pipeline that evaluates the quality of answers produced by the AjraSakha agricultural advisory bot against expert-validated answers from the Golden Database (GDB).

The pipeline scores each bot answer on four metrics:

- **Relevance** — Did the bot answer actually address what the farmer asked?
- **Faithfulness** — Did the bot stick to agricultural facts without hallucinating?
- **GDB Match** — How closely did the bot answer match the expert validated answer?
- **Agricultural Accuracy** — Did the answer mention the correct crop, treatment, and region?

---

## Why This Matters

A wrong pesticide dosage or wrong scheme name in an answer causes real harm to a farmer. Keyword-based pass/fail testing cannot catch this. This pipeline can.

## How to Run

### 1. Install dependencies

```
pip install groq python-dotenv
```

### 2. Set up your API key

Create a `.env` file in the project folder:

```
GROQ_API_KEY=your_groq_key_here
```

### 3. Run the evaluation

```
python evaluate.py
```

### 4. Generate the report

```
python report.py
```

## Current Limitations

This pipeline currently runs on mock data. To connect it to the real AjraSakha system:

1. Replace `mock_gdb.json` with real GDB entries
2. Replace the `simulate_bot_answer()` function with a call to the real AjraSakha API
3. Connect a Postgres database for persistent score storage instead of `results.json`
